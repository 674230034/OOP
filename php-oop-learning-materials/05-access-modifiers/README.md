# PHP OOP - การควบคุมการเข้าถึง (Access Modifiers)

**Access Modifiers** คือคีย์เวิร์ดที่ใช้กำหนด "สิทธิ์การเข้าถึง" พร็อพเพอร์ตี้ (Properties) และเมธอด (Methods) ภายในคลาส เพื่อความปลอดภัยของข้อมูล และป้องกันไม่ให้โค้ดส่วนอื่นภายนอกคลาสแอบเข้ามาแก้ไขข้อมูลสำคัญโดยไม่ได้รับอนุญาต (ตามหลักการ Encapsulation)

ใน PHP มี Access Modifiers หลัก ๆ อยู่ 3 ตัว:

1. **`public`** - เข้าถึงจากที่ไหนก็ได้ (ค่าเริ่มต้น)
2. **`protected`** - เข้าถึงได้จาก**ภายในคลาสตัวเอง** และ**คลาสที่สืบทอดไป (คลาสลูก)** เท่านั้น
3. **`private`** - เข้าถึงได้จาก**ภายในคลาสตัวเองเท่านั้น** คลาสลูกหรือภายนอกคลาสไม่มีสิทธิ์เข้าถึง

---

## 📊 ตารางสรุปสิทธิ์การเข้าถึง

| Access Modifier | ภายในคลาสตัวเอง | คลาสลูก (Inherited Class) | ภายนอกคลาส (Outside Class) |
| :--- | :---: | :---: | :---: |
| `public` | ✅ | ✅ | ✅ |
| `protected` | ✅ | ✅ | ❌ |
| `private` | ✅ | ❌ | ❌ |

---

## 🚀 ตัวอย่างโค้ดใช้งานจริง (Example)

ลองมาดูว่าจะเกิดอะไรขึ้นเมื่อเราพยายามเรียกใช้งานตัวแปรที่มีระดับสิทธิ์ต่างกันจากภายนอกคลาส:

> 📄 **ซอร์สโค้ดเต็ม:** คุณสามารถดูและทดลองรันโค้ดจริงได้ที่ไฟล์ [index.php](./index.php)

```php
<?php
class Car {
  public $brand;       // public
  protected $color;    // protected
  private $price;      // private
}

$myCar = new Car();

// 1. ลองเข้าถึงข้อมูลจาก "ภายนอกคลาส"

$myCar->brand = "Toyota"; //  ทำงานได้ปกติ (เพราะเป็น public)

// $myCar->color = "สีแดง"; //  Error! (เพราะเป็น protected)

// $myCar->price = 500000; //  Error! (เพราะเป็น private)
?>
