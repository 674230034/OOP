# บทที่ 1: PHP What is OOP? 🤔

**Object-Oriented Programming (OOP)** คือ รูปแบบการเขียนโปรแกรมเชิงวัตถุ ที่จะเปลี่ยนมุมมองการเขียนโค้ด จากเดิมที่มองเป็นลำดับขั้นตอน (Procedural) ให้กลายเป็นการมองทุกอย่างเป็น **"วัตถุ (Object)"** ที่มีชีวิตและหน้าที่ของตัวเอง

---

### 🧱 2 คำศัพท์สำคัญที่ต้องรู้!

เพื่อให้เข้าใจง่าย ให้จินตนาการถึง **"การสร้างรถยนต์"**

1. **Class (คลาส):** เปรียบเสมือน **"แบบแปลน หรือ พิมพ์เขียว (Blueprint)"** ของรถยนต์ที่อยู่บนกระดาษ มันยังใช้วิ่งไม่ได้ แต่มันบอกว่ารถคันนี้จะมีสีอะไร มีประตูกี่บาน และทำอะไรได้บ้าง
2. **Object (วัตถุ):** เปรียบเสมือน **"รถยนต์จริง ๆ"** ที่ถูกประกอบออกมาจากพิมพ์เขียวชิ้นนั้น พร้อมใช้งานและจับต้องได้

| คำศัพท์ใน OOP | เปรียบเทียบกับโลกจริง | หน้าที่ในโค้ด |
| :--- | :--- | :--- |
| **Properties** | ลักษณะของวัตถุ (เช่น สี, ยี่ห้อ, ขนาด) | ตัวแปร (Variables) ที่อยู่ในคลาส |
| **Methods** | พฤติกรรมของวัตถุ (เช่น สตาร์ทเครื่อง, เบรก, เร่งความเร็ว) | ฟังก์ชัน (Functions) ที่อยู่ในคลาส |

---

### 🌟 ข้อดีของการใช้ OOP

* **Reusable:** โค้ดสามารถนำกลับมาใช้ใหม่ได้ง่าย ลดการเขียนโค้ดซ้ำซ้อน
* **Maintainable:** จัดระเบียบโค้ดได้ดีขึ้น เมื่อระบบมีปัญหา สามารถเข้าไปแก้ไขเป็นส่วน ๆ ได้ง่าย ไม่พังไปทั้งระบบ
* **Scalable:** เหมาะมากสำหรับการพัฒนาโปรเจกต์ขนาดใหญ่ที่มีความซับซ้อน ร่วมกับนักพัฒนาหลายคน
